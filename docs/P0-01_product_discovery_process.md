# P0-01 — Product Discovery Process

This document defines the structured process we will follow to go from an existing Excel-based habit tracker to a fully specified Android app. Each phase produces one or more documents, numbered to reflect when they were created in the iteration cycle.

---

## Phases

### Phase 0 — Meta & Process
Define the discovery framework itself. Establish document structure and naming conventions.

**Documents produced:**
- `P0-01_product_discovery_process.md` ← this file

---

### Phase 1 — Scope Definition
Align on who the app is for, what it must do, and what stays out of v1. Output is a prioritized feature list and a clear MVP boundary.

**Documents produced:**
- `P1-01_scope_and_mvp.md`

---

### Phase 2 — Product Spec
Translate scope into detailed user stories, business rules, and edge cases for every feature in MVP.

**Documents produced:**
- `P2-01_user_stories.md`
- `P2-02_business_rules.md`

---

### Phase 3 — Design
Define the information architecture, screen flows, and visual design direction.

**Documents produced:**
- `P3-01_information_architecture.md`
- `P3-02_screen_flows_and_wireframes.md`
- `P3-03_design_system.md`

---

### Phase 4 — Implementation Planning
Finalize tech stack, data models, and break work into buildable milestones.

**Documents produced:**
- `P4-01_tech_stack_decisions.md`
- `P4-02_data_model.md`
- `P4-03_build_milestones.md`

---

## Document Naming Convention

```
P{phase}-{sequence}_{short_title}.md
```

Example: `P1-01_scope_and_mvp.md` = Phase 1, first document, about scope and MVP.
